# 04 - Data Driven Testing ## Overview This collection demonstrates data-driven testing (DDT) methodology using external data sources to parameterize API tests. It showcases how to execute the same test scenario multiple times with different input data from CSV or JSON files, enabling comprehensive test coverage with minimal code duplication. This approach is essential for testing multiple scenarios, edge cases, and validating API behavior across various data sets. ## API Used - **Base URL**: [https://fakestoreapi.com](https://fakestoreapi.com) - **Documentation**: [https://fakestoreapi.com/docs](https://fakestoreapi.com/docs) - **Endpoint**: POST /products - **No Authentication Required**: Public testing API ## What is Data Driven Testing? Data-driven testing is a methodology where test data is separated from test logic, allowing the same test to run multiple times with different inputs. Benefits include: - ‚úÖ Reduced code duplication - ‚úÖ Easy test data management - ‚úÖ Comprehensive coverage with multiple scenarios - ‚úÖ Quick validation of edge cases - ‚úÖ Scalable test maintenance - ‚úÖ Non-technical stakeholders can manage test data ## Features Demonstrated - ‚úÖ Parameterized testing with external data - ‚úÖ CSV file integration - ‚úÖ JSON file integration - ‚úÖ Variable substitution from data files - ‚úÖ Iteration-based testing - ‚úÖ Bulk test execution - ‚úÖ Dynamic data injection ## Collection Contents ### 1\. Advanced Data-Driven Testing Techniques ### 1\. CSV with Special Characters ``` csv title,price,description,image,category "Product with, comma",99.99,"Description with ""quotes""",url,category ``` ### 2\. Conditional Testing Based on Data ``` javascript pm.test("Validate price range", function() { const price = parseFloat(pm.iterationData.get("price")); if (price > 100) { // Additional validation for expensive items pm.expect(json).to.have.property('warranty'); } }); ``` ### 3\. Dynamic Test Names ``` javascript pm.test(`Created product: ${pm.iterationData.get("title")}`, function() { pm.response.to.have.status(201); }); ``` ### 4\. Iteration-Specific Variables ``` javascript // Access current iteration data const currentTitle = pm.iterationData.get("title"); const currentPrice = pm.iterationData.get("price"); console.log(`Testing: ${currentTitle} at $${currentPrice}`); ``` ## Use Cases for Data-Driven Testing ### 1\. Bulk Product Creation Test creating multiple products with varied attributes: - Different categories (electronics, clothing, furniture) - Various price ranges ($1 - $10,000) - Different description lengths - Multiple image formats ### 2\. Edge Case Testing ``` csv title,price,description,image,category "",99.99,Valid description,url,electronics Valid Title,0,Valid description,url,electronics Valid Title,-10,Valid description,url,electronics Valid Title,99.99,"",url,electronics ``` ### 3\. Category Validation Test all available categories: ``` csv title,price,description,image,category Product 1,99.99,Description,url,electronics Product 2,99.99,Description,url,jewelery Product 3,99.99,Description,url,men's clothing Product 4,99.99,Description,url,women's clothing ``` ### 4\. Performance Testing Create hundreds of products to test API performance: ``` bash # Generate CSV with 100 rows, then run newman run collection.json \ --iteration-data data/bulk-products.csv \ --delay-request 100 ``` ## Best Practices ### Data File Management 1. **Keep data files version controlled** (Git) 2. **Use descriptive file names**: `products-valid.csv`, `products-invalid.csv` 3. **Document expected results** in README 4. **Separate positive and negative test data** ### Column/Key Naming 1. **Match variable names exactly** (case-sensitive) 2. **Use consistent naming convention** (snake_case or camelCase) 3. **Avoid special characters** in column names 4. **Keep names descriptive**: `product_title` not just `title` ### Test Data Design 1. **Include boundary values** (min, max, zero) 2. **Test special characters** (quotes, commas, unicode) 3. **Mix valid and invalid data** (in separate files) 4. **Cover all categories/types** 5. **Include realistic data** for demos ### Error Handling ``` javascript pm.test("Handle missing data gracefully", function() { const title = pm.iterationData.get("title"); if (!title) { console.warn("Warning: title is missing in data row"); } // Continue with tests... }); ``` ## Troubleshooting ### Issue: Variables Not Substituting **Problem**: `{{title}}` appears as literal text in request **Solution**: - Verify column names match variable names exactly - Check CSV file has no extra spaces in headers - Ensure data file is properly selected in Runner ### Issue: Test Fails on Some Iterations **Problem**: Some data rows cause failures **Solution**: ``` javascript // Add iteration info to console console.log(`Iteration ${pm.info.iteration} - Testing: ${pm.iterationData.get("title")}`); ``` ### Issue: CSV Not Parsing Correctly **Problem**: Commas in data break parsing **Solution**: Wrap fields in quotes: ``` csv "Product with, comma","99.99","Description, with commas" ``` ## Prerequisites - Postman Desktop App or Postman Web - Data file (CSV or JSON format) - Internet connection - No API key required ## Skills Showcased - Data-driven testing methodology - External data file integration (CSV, JSON) - Parameterized testing - Iteration-based test execution - Variable substitution from files - Bulk test data management - Newman CLI data integration - Test scalability techniques - Edge case validation - Performance testing preparation - Test data organization - Report generation with iterations ## Folder Structure for Data Files ``` project/ ‚îú‚îÄ‚îÄ collections/ ‚îÇ ‚îî‚îÄ‚îÄ 04-Data-Driven-Testing.postman_collection.json ‚îú‚îÄ‚îÄ data/ ‚îÇ ‚îú‚îÄ‚îÄ products.csv ‚îÇ ‚îú‚îÄ‚îÄ products.json ‚îÇ ‚îú‚îÄ‚îÄ products-valid.csv ‚îÇ ‚îú‚îÄ‚îÄ products-invalid.csv ‚îÇ ‚îî‚îÄ‚îÄ products-bulk.csv ‚îú‚îÄ‚îÄ reports/ ‚îÇ ‚îî‚îÄ‚îÄ ddt-report.html ‚îî‚îÄ‚îÄ README.md ``` ## Real-World Applications 1. **E-commerce**: Test product catalog uploads 2. **User Registration**: Test bulk user creation 3. **Data Migration**: Validate migrated data 4. **Regression Testing**: Run same tests with different data sets 5. **Load Testing**: Generate multiple requests with varied data 6. **Compliance Testing**: Validate required fields across data sets ## Extending This Collection ### Add More Test Scenarios ``` javascript // Validate price format pm.test("Price is numeric", function() { const price = pm.iterationData.get("price");
