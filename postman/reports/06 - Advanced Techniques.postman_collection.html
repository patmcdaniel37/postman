Schema validation ensures the API returns data in the expected format, type, and structure. It's a form of **contract testing** that catches breaking changes early. #### Implementation Details ``` javascript // Define expected schema const productSchema = { type: "object", required: ["id", "title", "price", "description", "category", "image", "rating"], properties: { id: { type: "number" }, title: { type: "string" }, price: { type: "number" }, description: { type: "string" }, category: { type: "string" }, image: { type: "string" }, rating: { type: "object", required: ["rate", "count"], properties: { rate: { type: "number" }, count: { type: "number" } } } } }; ``` #### AJV Library Usage ``` javascript const Ajv = require('ajv'); // Import JSON validator const ajv = new Ajv({ logger: console }); // Create validator instance const validate = ajv.compile(productSchema); // Compile schema const valid = validate(response); // Validate response ``` #### Tests Performed - ‚úÖ Status code validation (200) - ‚úÖ Response is an object - ‚úÖ Schema validation with AJV - ‚úÖ All required fields present - ‚úÖ All data types correct (number, string, object) - ‚úÖ Nested object validation (rating.rate, rating.count) - ‚úÖ Error logging for schema violations **Benefits**: - Catches API contract violations - Validates data types automatically - Ensures required fields are present - Documents expected API structure - Prevents breaking changes ### 2\. Conditional Validation **Purpose**: Adapt test validation based on response status codes #### What is Conditional Validation?
